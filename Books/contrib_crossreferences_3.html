  • Books

    Documentation for the IOP Publishing specific implementation of BITS 2.0 for book content

    • Root element

      The root element for a book is a <book> element.

    • Book Meta

      <book-meta >is a mandatory element for all books as a container for the book level metadata.

    • Front Matter

      Capture the front matter information for a book inside a <front-matter> element.

    • Chapters

      All of a books chapters must be captured in seperate <book-part > elements within a <book-body> element.

      • Book-Part Meta

        <book-part-meta>is a mandatory element for all chapters as a container for the chapter level metadata.

        • Book Part ID

          All chapters must contain a <book-part-id> element containing the chapter level doi.

        • Chapter Title

          Capture the chapter title in a <title> element inside a <title-group> element.

        • Chapter Contributors

          Chapter level contributor information is captured in a <contrib-group> element.

          • Authors and Editors

            Capture each author or editor contributor in a <contrib> element child of <contrib-group>

          • Collaborations

            Capture a collaboration group in a <contrib> element child of <contrib-group>

          • Contrib Xrefs

            Contributors with any associated affiliation, footnote or biography information must have that information explictly linked using an <xref> element.

          • Affiliations

            Capture affiliations in <aff> elements.

        • Page Numbers

          Capture first and last page numbers for a chapter in fpage and lpage elements respectively.

        • Chapter Self-uris

          Capture the filenames of the PDF, EPUB and MOBI representations of the chapter in self-uri elements.

        • Abstract

          Capture Abstracts in a <abstract> element.

      • Chapter Body

        Capture the body content of a book chapter inside a <body> element as a child of the relevent <book-part>.

Contrib Xrefs

Contributors with any associated affiliation, footnote or biography information must have that information explictly linked using an <xref> element.

Each <xref> element must have a @ref-type attribute with the value set depending on what is being linked to. The allowed values are:
Table 1. @ref-typeAllowed Values
Xref Target @ref-type Value
Affiliation aff
Biography bio
Footnote fn
Correspondence corresp

Each <xref> element must have a @rid attribute which contains the identifer for the target of the xref.

If the target of the <xref> has a <label> element then the text of that label must also be included as the text content of the <xref>.

Example

<contrib contrib-type="author">
  <name name-style="western">
    <surname>McGurn</surname>
    <given-names>Arthur R</given-names>
  </name>
  <xref ref-type="aff" rid="aff1"/>
  <xref ref-type="bio" rid="bio1"/>
</contrib>
